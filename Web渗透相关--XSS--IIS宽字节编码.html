IIS宽字节编码


asp+iis的环境中，当我们请求的url中存在单一的百分号%时，iis+asp会将其忽略掉
例如sel%ect user f%rom table

iis支持unicode的解析，当我们请求的url存在unicode字符串的话iis会自动将其转换

例如%u0031等


unicode在iis解析之后会被转换成multibyte，但是转换的过程中可能出现：
多个widechar会有可能转换为同一个字符。

譬如select中的e对应的unicode为%u0065，但是%u00f0同样会被转换成为e。
